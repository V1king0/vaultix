Security Model

Three domains. One clear mental model.

Vaultix is engineered around a color-coded trust model. You'll always know the security context of your active window without complex manual configuration.

Green
Trusted Domain

Reserved for fully trusted tools — SSH clients, password managers, admin utilities. Full network and filesystem access. The core "vault" of your digital identity.

Full network Home dir Persistent Clipboard ✓
Amber
Normal Domain

Daily use — web browsing, email, development. Network access permitted, home directory limited, inter-domain sharing requires explicit confirmation.

Network Limited home Persistent Prompt share
Red
Isolated Domain

High-risk activities — unknown files, suspicious sites. No host access. No persistent data. Traffic routed through Tor only. Container destroyed on exit.

Tor only No home dir Ephemeral No clipboard

Application Layer
Each app → own Podman container
Each container → own Wayland surface
Each container → own AppArmor profile
Each container → own network namespace
Rootless Podman
Containers are unprivileged child processes of your user session. There is no central root daemon, drastically reducing the attack surface.
Wayland Isolation
Native Wayland surfaces prevent cross-application keylogging and screenshotting—structural weaknesses found in legacy X11 systems.
AppArmor Profiles
Every application runs under a specific MAC profile. Any action not explicitly white-listed is blocked by the kernel.
Controlled Data Flow
Vaultix will utilize a policy-governed confirmation prompt for all clipboard and file transfers between domains to prevent leaks.
Core Features

Built secure. Not patched secure.

Development is focused on ensuring security is a structural constraint, not an option.

01
🔒
Full-Disk Encryption

LUKS encryption integration is a primary target, protecting data-at-rest from the first boot.

02
📦
Container-per-App

The OS core is designed to treat every application as an isolated Podman instance.

03
🖥
Wayland Native

No X11 compatibility layer. A pure Wayland environment ensures display bus isolation.

04
🛡
AppArmor MAC

Kernel-enforced mandatory access control per container is being implemented for every domain.

05
🧅
Tor-by-Default (Red)

Network isolation for the Red domain will route all traffic exclusively through the Tor network.

06
♻️
Ephemeral Containers

Disposable red domain containers are designed to be destroyed on exit, leaving zero persistent artifacts.

Future Roadmap

Post-Alpha vision.

Planned enhancements that extend Vaultix's core philosophy once the initial distribution stabilizes.

01
Verified Boot Chain

UEFI Secure Boot with user-controlled keys + dm-verity to detect offline tampering before execution begins.

02
Immutable Root Filesystem

A read-only root with an ephemeral overlay, ensuring the host layer remains forensically clean every boot.

03
Hardware Security Key

FIDO2/YubiKey requirements for LUKS unlock and domain escalation to block remote pivoting.

04
Container Attestation

Verification of container images against signed manifests before launch to prevent supply-chain attacks.
